# Contributing to SubstrateMC

Thank you for your interest in contributing! This project is currently in **alpha**, so your feedback and contributions are very welcome.

## How to Contribute

### Reporting Bugs

The easiest way to help is by reporting bugs.

- If you find a bug or unexpected behavior, please open an [issue](https://github.com/benjamin-l-Renz/SubstrateMC/issues).
- Include steps to reproduce, what you expected, and what happened.

### Feature Requests

- New feature ideas are welcome.

### Pull Requests

1. Fork the repository.
2. Create a new branch for your feature or bug fix.
3. Make your changes and commit them.
4. Push your branch to your fork.
5. Open a pull request from your branch to the main repository.
6. Ensure your code follows the project's style guide and passes all tests.
7. Wait for feedback and address any requested changes.

## Style Guidelines

To keep the codebase consistent and readable, please follow these guidelines:

- Format your code using `cargo fmt`
- Fix any linting errors using `cargo clippy`
- Ensure your code is well-documented with comments and docstrings
- Use snake_case for variable and function names
- Keep lines under 100 characters
- Use meaningful commit and pull request messages
- Use PascalCase for struct and enum names
- Use SCREAMING_SNAKE_CASE for constants
